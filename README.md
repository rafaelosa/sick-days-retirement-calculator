# מחשבון אומדן מענק ימי מחלה בפרישה

מחשבון אינטרנטי בעברית (RTL) לחישוב אומדן מענק ימי מחלה בפרישה לעובדי מדינה.

הנתונים נשמרים אוטומטית ב-**Google Sheet** (כמסד נתונים), ובמקביל נשלח **מייל אוטומטי** עם פרטי הפנייה.

פותח על ידי **אלפי ביטוח ופיננסים** — מומחי פרישה וטופס 161.

# התקנה — שלב אחר שלב

ההתקנה מתחלקת ל-2 חלקים:
- **חלק א**: הקמת ה-Backend (Google Sheet + Apps Script) — 10 דקות
- **חלק ב**: העלאת ה-Frontend ל-GitHub Pages — 5 דקות

---

## חלק א — הקמת ה-Backend

### שלב 1: יצירת Google Sheet

1. היכנס ל-[sheets.google.com](https://sheets.google.com) (חשבון Google רגיל מספיק)
2. לחץ על **+** ליצירת גיליון חדש
3. תן לו שם, לדוגמה: `מחשבון מענק ימי מחלה — הגשות`

### שלב 2: הוספת ה-Apps Script

1. בתוך ה-Sheet, לחץ על **Extensions** → **Apps Script** (התפריט באנגלית — הרחבות ← Apps Script)
2. יפתח חלון חדש עם עורך קוד. מחק את כל הקוד הקיים שם
3. פתח את הקובץ `apps-script.gs` מהפרויקט הזה והעתק את כל תוכנו
4. הדבק את הקוד בעורך
5. בראש הקובץ, ודא ש-`RECIPIENT_EMAIL` מוגדר לכתובת המייל שלך:
   ```javascript
   const RECIPIENT_EMAIL = 'office@alafi-ins.co.il';
   ```
6. שמור (Ctrl+S או File → Save). תן לפרויקט שם, לדוגמה: `Sick Days Calculator Webhook`

### שלב 3: הרצת בדיקה (חשוב!)

1. בעורך הקוד, בחר את הפונקציה `testSubmission` מהתפריט הנפתח שליד כפתור ה-Run
2. לחץ **Run** (▶)
3. בפעם הראשונה Google ידרוש הרשאות:
   - יופיע "Authorization required" → לחץ **Review permissions**
   - בחר את חשבון Google שלך
   - יופיע מסך אזהרה "Google hasn't verified this app" — זה תקין כי זה הסקריפט הפרטי שלך:
     - לחץ **Advanced**
     - לחץ **Go to [שם הפרויקט] (unsafe)** — זה לא באמת unsafe, זה הסקריפט שלך
   - אשר את ההרשאות (גישה ל-Sheets ולשליחת מיילים)
4. אם הכל עבד: ייווצר גיליון חדש בשם "הגשות" עם שורת בדיקה, ותקבל מייל בדיקה לתיבה שלך

### שלב 4: פרסום כ-Web App

1. בעורך הקוד, לחץ **Deploy** (פינה ימנית עליונה) → **New deployment**
2. ליד "Select type" לחץ על גלגל השיניים ⚙ ובחר **Web app**
3. מלא:
   - **Description**: `Sick Days Calculator v1`
   - **Execute as**: `Me (האימייל שלך)`
   - **Who has access**: `Anyone` ⚠ חשוב — בלי זה המחשבון לא יוכל לשלוח נתונים
4. לחץ **Deploy**
5. שוב יבקש הרשאות — אשר
6. **תקבל URL** שנראה כך:
   ```
   https://script.google.com/macros/s/AKfycbx...../exec
   ```
7. **העתק את ה-URL הזה — תזדקק לו בשלב הבא**

> **חשוב**: אם תעדכן את הקוד בעתיד, תצטרך **Manage deployments → Edit (עיפרון) → New version → Deploy** כדי שהשינויים ייכנסו לתוקף.

## חלק ב — חיבור המחשבון ל-Backend

### שלב 5: עדכון ה-URL ב-`index.html`

1. פתח את `index.html` בעורך טקסט (Notepad / VS Code / כל עורך)
2. חפש את השורה (בערך שורה 950):
   ```javascript
   const APPS_SCRIPT_URL = 'PASTE_YOUR_GOOGLE_APPS_SCRIPT_URL_HERE';
   ```
3. החלף את הטקסט בתוך המירכאות ב-URL שהעתקת בשלב 4:
   ```javascript
   const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/AKfycbx...../exec';
   ```
4. שמור את הקובץ

### שלב 6: העלאה ל-GitHub Pages

1. צור Repository חדש ב-GitHub כ-**Public**
2. העלה את הקבצים: `index.html`, `README.md`, `.gitignore`, `LICENSE.txt`
3. **אל תעלה** את `apps-script.gs` (אם תרצה לשמור אותו ב-Repo, זה בסדר — הוא לא מריץ שום דבר ב-Pages)
4. הפעל GitHub Pages: **Settings → Pages → Branch: main → Save**
5. תוך 1-2 דקות המחשבון יהיה זמין ב-`https://[שם-משתמש].github.io/[שם-מאגר]/`

### שלב 7: בדיקה סופית

1. פתח את כתובת ה-Pages
2. מלא את הטופס בנתוני בדיקה
3. סמן את שתי תיבות הסימון
4. לחץ "חשב מענק"
5. לחץ "שלח נתונים"
6. אמור להופיע: "הנתונים נשלחו בהצלחה. תודה!"
7. בדוק את ה-Google Sheet — שורה חדשה
8. בדוק את המייל — מייל מעוצב חדש

## שינוי כתובת המייל לקבלת ההתראות

ערוך את `apps-script.gs`, שנה את:
```javascript
const RECIPIENT_EMAIL = 'office@alafi-ins.co.il';
```
ועשה **Deploy → Manage deployments → Edit → New version → Deploy** כדי לעדכן.

## הוספת שדות נוספים בעתיד

אם תרצה להוסיף שדה חדש לטופס:
1. הוסף את השדה ב-`index.html` (בקטע ה-form)
2. הוסף אותו ל-`payload` ב-fetch
3. הוסף אותו לכותרות ולשורה ב-`apps-script.gs` (בפונקציה `saveToSheet`)
4. רעדכן את הסקריפט (Deploy → New version)

## פתרון בעיות נפוצות

| בעיה | פתרון |
|------|-------|
| "המערכת עדיין לא מחוברת לשרת" | לא הוחלף ה-`APPS_SCRIPT_URL` ב-`index.html` |
| הנתונים לא מגיעים ל-Sheet | ודא שה-Web App פורסם עם "Anyone access" |
| לא מתקבל מייל | בדוק תיקיית ספאם; ודא שה-`RECIPIENT_EMAIL` נכון |
| הודעת שגיאה בשליחה | בדוק ב-Apps Script: View → Executions → תראה את השגיאה |
